# Working Agreement – Gatekeeper Capstone Project

## 1. Purpose
This working agreement defines how the Gatekeeper team will collaborate, communicate, and deliver work effectively throughout the capstone project.

---

## 2. Communication
- Primary communication will occur via WhatsApp or Teams.
- Weekly group meetings will be held to discuss progress and blockers.
- Important decisions and updates will be documented in GitHub Issues or Project notes.
- Team members are expected to respond to messages within 24 hours.

---

## 3. Roles & Responsibilities
- Each member is responsible for tasks aligned with their assigned role.
- Tasks will be selected from the product backlog and tracked using GitHub Projects.
- Members are expected to complete assigned tasks within agreed timelines.
- If a member is blocked, they must notify the group as early as possible.

---

## 4. Version Control & GitHub Usage
- GitHub will be used for all version control and project tracking.
- The `main` branch represents stable and reviewed code only.
- All significant changes must be made through feature branches.
- Pull requests should be reviewed by at least one team member before merging.
- Commit messages must be clear and descriptive.

---

## 5. Quality & Collaboration
- Code and documentation should follow agreed formatting and style conventions.
- Security and privacy considerations must be respected in all features.
- Team members should support and review each other’s work respectfully.

---

## 6. Conflict Resolution
- Conflicts should be discussed openly within the team first.
- If unresolved, the issue will be escalated to the course instructor.
- Decisions will prioritize project success and fairness.

---

## 7. Attendance & Accountability
- All members are expected to attend scheduled meetings.
- If a member cannot attend, advance notice must be given.
- Repeated lack of participation will be documented and reported if necessary.
